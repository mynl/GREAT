great.utils.binary(num)

Certain floats like 0.1 have very awkward binary reps and lead to floating point issues, and hence indexing issues. Best just to avoid. E.g. don’t use 0.1 use 0.125 as a bs. etc.

This function lets you see the binary expansion of the float.

Struct can provide us with the float packed into bytes. The ‘!’ ensures that it’s in network byte order (big-endian) and the ‘f’ says that it should be packed as a float. Alternatively, for double-precision, you could use ‘d’.

-i, --ignore

-n, --new_slide

Set to suppress new slide.

-o, --option

Set option for tables that vary with options, uses decorated top_value filename.

-p, --promise

first and last thick rules by default others below (Python, zero-based) row number, excluding title row

Rules

  • hrule at i means below row i of the table. (1-based) Top, bottom and below index lines are inserted automatically. Top and bottom lines are thicker.

sparsify number of cols of multi index to sparsify

  1. with_table : all output in @@@ file and include in main md file; use when caption is generic

  2. caption: puts caption text in the main markdown file, use when caption will be edited

  3. inline: all output in md file directly (not recommended)

font_size = scriptsize, footnotesize etc.

label and columns have _ escaped for TeX but the caption is not - so beware!

multipart=True for widetable, then skip the beamer caption
